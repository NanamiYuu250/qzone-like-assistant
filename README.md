## 主要功能

- 在 App 内打开腾讯官方 QQ 空间页面，支持扫码或页面提供的其他登录方式。
- 监听好友动态，并按照扫描间隔与点赞冷却执行操作。
- 可按设定周期自动刷新页面；刷新前后仅使用动态稳定 ID，并按主页面/frame 上下文分别匹配顶部锚点。
- 可设置每日尝试上限、启动回扫上限和关键词过滤。
- 启动时先登记页面已有内容，尽量避免把首屏历史动态误认为刚发布的新动态。
- 可选“启动回扫”，处理开启监听前已经加载的少量动态。
- 登录状态保存在本机 WebView2 用户目录，正常关闭再打开时通常无需重新输入账号密码。
- 停止监听或重启程序不会清除剩余冷却时间和当天额度。

## 下载与安装

1. 打开 [Releases](https://github.com/NanamiYuu250/qzone-like-assistant/releases/latest)。
2. 下载名称中包含 `win-x64.zip` 的压缩包。
3. 将压缩包完整解压到一个固定文件夹。
4. 运行文件夹内的 `QQ空间点赞助手.exe`。
5. 如需桌面入口，双击 `创建桌面快捷方式.cmd`；脚本会按当前解压目录创建快捷方式。

从 GitHub 下载的是 ZIP 发布包，因此第一次使用需要解压；不要只把 EXE 单独拖出压缩包，旁边的 DLL 和运行文件也必须保留。

运行要求：Windows 10/11、Microsoft Edge WebView2 Runtime。Windows 11 通常已经自带 WebView2 Runtime。

## 使用方法

1. 在右侧腾讯页面完成登录，并进入“好友动态”。
2. 在左侧设置扫描间隔、点赞冷却、每日上限和回扫选项。
3. 点击“开启自动点赞”。
4. 不需要时点击“停止”；如需删除本机登录状态，点击“退出并清除登录”。

自动点赞依赖本地 WebView2 页面，关闭软件或结束后台进程后不会继续运行。若需要全天在线，必须让程序保持运行；本项目目前不提供服务器托管功能。

## 建议参数

下面是偏保守的个人账号建议，并非平台公布的安全线：

| 设置 | 建议值 | 说明 |
| --- | ---: | --- |
| 扫描间隔 | 3～5 秒 | 检查页面更新的频率，不等于点赞频率 |
| 自动刷新 | 开启，5 分钟 | 刷新前后安全对比顶部动态锚点 |
| 点赞冷却 | 8～15 秒 | 建议从 10 秒开始 |
| 每日上限 | 30～100 次 | 建议先设为 60 次 |
| 启动回扫 | 按需开启 | 不需要补赞旧动态时建议关闭 |
| 回扫点赞上限 | 5～10 次 | 同时受每日额度的 30% 限制 |

不建议将每日上限设置为数百或数千。输入值只有在点击“开启自动点赞”时才会读取并保存；超出程序范围的数值会被自动限制到允许范围。

## “成功”和“尝试”的区别

- **尝试数**：程序已经对页面中的点赞控件发出一次真实点击。
- **成功数**：点击后，页面进一步确认该动态进入“已赞”状态。
- 每日上限按照**尝试数**扣减，而不是只按照成功数扣减。
- 未确认成功的点击仍占用尝试额度，但不会加入已处理去重记录。
- 清除登录状态不会清空当天成功数或尝试数。

这种计数方式可以避免页面请求失败或确认超时时，通过反复点击绕过每日额度。

## 登录与本地数据

- 程序不读取或保存 QQ 明文密码；账号和密码输入发生在腾讯页面内。
- 登录 Cookie 和页面数据仅保存在 `%LocalAppData%\QzoneLikeAssistant\WebView2`。
- 设置和当天统计保存在 `%LocalAppData%\QzoneLikeAssistant`。
- 点击“退出并清除登录”会删除该 App 保存的 Cookie 和浏览数据，但不会清空当天额度。

如果腾讯主动使登录失效、要求重新验证，或用户清除了登录数据，仍需要重新登录。

## 工作边界

- 不调用私有点赞接口，只操作当前页面已经加载的控件。
- 不绕过验证码、登录验证或平台风控。
- 页面结构更新后，自动识别可能暂时失效，需要更新程序适配。
- 启动回扫最多占每日额度的 30%，并有单独的回扫上限。
- 点击停止、清除登录或关闭窗口后，当前自动化会话会失效。

## 开发与构建

需要 .NET 8 SDK 和 Microsoft Edge WebView2 Runtime。

```powershell
dotnet restore
dotnet run
```

发布 Windows x64 版本：

```powershell
dotnet publish -c Release -r win-x64 --self-contained false -o publish
```

运行烟雾测试（需要 Node.js）：

```powershell
dotnet run --project tests/QzoneLikeAssistant.SmokeTests -c Release
```

运行 WebView2 本地集成测试（仅加载测试 HTML，不访问 QQ）：

```powershell
dotnet run --project tests/QzoneLikeAssistant.WebView2IntegrationTests -c Release
```

## 版本说明

### V1.10.1

- 修复启动回扫只处理顶部少量动态后提前停止的问题，支持动态 iframe 与外层页面滚动区分离的 QQ 空间布局。
- 刷新或页面 frame 替换时保留未完成的回扫进度、额度和原截止时间，不会因刷新清空或重新计算。
- 回扫期间跨主页面与 frame 同步状态；慢加载的历史条目只登记为历史内容，不会在回顶后误判为新动态。
- 加强页面替换、停止后立即重启、回扫截止时间和错误滚动容器的安全清理。

### V1.10

- 增加可关闭的安全自动刷新，默认每 5 分钟执行一次。
- 刷新前按页面上下文分别保存稳定 ID 锚点，刷新后只把唯一匹配的旧锚点之前条目判定为新增。
- 锚点少于两个、匹配有歧义或找不到旧锚点时，当前上下文全部按历史内容保护，避免误赞。
- 页面导航期间暂停扫描，加载完成并重建基线后恢复。

### V1.9

- 更换为淡粉与白色分区界面。
- 更新应用图标和发布包图标。
- 保留 V1.8 的额度、冷却、停止会话和动态基线保护。

### V1.8

- 完善异步点赞结果确认与尝试额度统计。
- 改进 frame 重建、启动基线和分批加载历史动态的处理。
- 冷却时间持久化，停止监听或重启程序不能绕过冷却。
