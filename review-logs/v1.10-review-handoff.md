## 本轮修复

1. 刷新锚点按主页面/frame 上下文分别保存和应用，不再把所有 key 合并后广播到每个上下文。
2. 新增 `RefreshContextMatcher`：至少需要两个共同稳定键，并且当前与旧上下文必须形成唯一的一对一匹配；主页面不允许匹配 frame，歧义或不明确时按无锚点处理。
3. `BuildCaptureRefreshKeys` 与 `BuildApplyAndArmRefreshKeys` 只接受 `data-tid`、`data-feedskey`、`data-unikey` 或 `id`，删除 `innerText` fallback。
4. 新增 `NavigationGeneration`，每次允许的 `NavigationStarting` 都记录 `NavigationId` 并重启 watchdog；只有最新导航的 Completed/OperationCanceled 可以清除扫描暂停，旧导航完成事件被忽略。
5. 设置迁移按源版本处理：仅 `<5` 执行旧默认值迁移，V5→V6 保留用户自定义的 2/8/30；SmokeTests 已覆盖。
6. 刷新差异与 tracker arm 合并到同一段同步 JavaScript 中；无锚点也在同一次调用中完成 arm，消除两个异步阶段之间的漏赞窗口。

## 新增/更新测试

- V5 自定义设置迁移保持。
- frame 重排后的上下文匹配、单锚点拒绝、重复候选拒绝、主页面/frame 跨上下文拒绝。
- 导航代次：旧完成事件不能清最新状态，最新 OperationCanceled 可解除暂停。
- 刷新差异与 arm 原子完成；无锚点路径同样完成 arm 且新增数为 0。
- 刷新脚本不包含 `innerText` fallback。
- 真实 WebView2 本地 HTML 用例确认：无稳定 ID 的正文不会生成刷新锚点。
